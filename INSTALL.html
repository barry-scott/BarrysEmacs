Installing Barry's Emacs from source

Ready to install kits for Windows or macOS (Mac OS X) are available from:

http://www.barrys-emacs.org/emacs8.html.

Dependencies

Linux and *BSD systems may package some of the dependencies. In which case these will be used.

For Windows and macOS all dependencies will need to be installed.

For most platforms you will need:

  • git to clone the Barry's Emacs sources
  • Python 3.10 or newer, 3.11 recommended from https://python.org
  • python PyQt6 from PyPI- pip install --user PyQt6
  • pycxx 7.1.4 from https://sourceforge.net/projects/cxx/
  • sqlite-amalgamation-3280000.zip from https://sqlite.org/download.html
  • hunspell-1.7.0.tar.gz from https://github.com/hunspell/hunspell/tags
  • hunspell-en_AU-large-2019.10.06.zip from http://wordlist.aspell.net/dicts/
  • hunspell-en_CA-large-2019.10.06.zip from http://wordlist.aspell.net/dicts/
  • hunspell-en_GB-large-2019.10.06.zip from http://wordlist.aspell.net/dicts/
  • hunspell-en_US-large-2019.10.06.zip from http://wordlist.aspell.net/dicts/

Use the Builder/install-dependencies.sh script to install the dependencies into the Imports folder

install-dependencies.sh tailors the install based on the operating system its detects.

Linux

Note: For the purposes of this documentation it is assumed that the Barry's Emacs files are in ~/Projects/BarrysEmacs, however you can replace that path with any folder name you like.

  1. Fetch source
  2. mkdir -p ~/Projects
  3. cd ~/Projects
  4. git clone https://github.com/barry-scott/BarrysEmacs.git
  5. Build GUI and CLI versions of bemacs
  6. cd BarrysEmacs/Builder
  7. . ./builder_custom_init.sh
  8. ./build-linux.sh
  9. Test the CLI build.
  10. cd ../Editor
  11. ./bemacs-cli.sh
  12. Test the GUI build.
  13. cd ../Editor/PyQtBEmacs
  14. ./bemacs-linux.sh --noredirect
  15. Install into /usr/local (sudo will prompt for your password).
  16. ./local-install-linux.sh

To create RPMs using mock run ./build-for-fedora.sh mock-testings

To create DEBs review ./build-for-debian.sh and adapt for your needs.

NetBSD

Note: For the purposes of documentation it is assumed that the Barry's Emacs files are in ~/Projects/BarrysEmacs, however you can replace that path with any folder name you like.

Note: Linux versions of scripts are used where they are compatible with NetBSD.

  1. Fetch sources
  2. mkdir -p ~/Projects
  3. cd ~/Projects
  4. git clone https://github.com/barry-scott/BarrysEmacs.git
  5. Build GUI and CLI versions of bemacs
  6. cd BarrysEmacs/Builder
  7. . ./builder_custom_init.sh
  8. Download dependencies into a dependencies-folder
  9. ./install-dependencies.sh <dependencies-folder>
  10. ./build-netbsd.sh

Test the CLI build.

  1. cd ../Editor
  2. ./bemacs-cli.sh

Install into /usr/local (sudo will prompt for your password).

  1. ./local-install-linux.sh

Windows

Note: For the purposes of documentation it is assumed that the Barry's Emacs files are in %USERPROFILE%\Projects\BarrysEmacs, however you can replace that path with any folder name you like.

These are the Windows specific dependencies

  • git - https://git-for-windows.github.io
  • Python 3.11 win64 or newer strongly recommended.
  • Visual C++ 2017 Community Edition (Version 14.1).
  • py -3.11 -m pip install --user --upgrade win-app-packager
  • py -3.11 -m pip install --user --upgrade PyQt6
  • inno setup - from http://www.jrsoftware.org/isinfo.php
  1. Fetch sources
  2. cd %USERPROFILE%
  3. mkdir Projects
  4. cd Projects
  5. git clone https://github.com/barry-scott/BarrysEmacs.git
  6. Build GUI version of bemacs
  7. cd BarrysEmacs\Builder
  8. builder_custom_init.cmd
  9. build-windows.cmd

Test the GUI build.

  1. cd ..\Editor\PyQt6
  2. bemacs-windows.cmd --noredirect

macOS

Note: For the purposes of documentation it is assumed that the Barry's Emacs files are in ~/Projects/BarrysEmacs, however you can replace that path with any folder name you like.

The build will create a DMG that contains the Barry's Emacs App.

  • Install XCode from the macOS App store
  • Run Xcode and allow it to install additional components.
  • Install the XCode command line tools - xcode-select --install
  • Download dependencies into a dependencies-folder
  1. Fetch source
  2. mkdir -p ~/Projects
  3. cd ~/Projects
  4. git clone https://github.com/barry-scott/BarrysEmacs.git
  5. Build GUI and CLI versions of bemacs
  6. cd BarrysEmacs/Builder
  7. . ./builder_custom_init.sh
  8. ./install-dependencies.sh <dependencies-folder>
  9. ./build-macosx.sh

Test the GUI build.

  1. cd ../Editor
  2. ./bemacs-gui.sh --noredirect

Test the CLI build.

  1. cd ../Editor
  2. ./bemacs-cli.sh --noredirect
